**mcpcn** is a shadcn-compatible registry of compound React components for [MCP Apps](https://modelcontextprotocol.io/). Its visual patterns start from [Manifest UI](https://github.com/mnfst/manifest-ui), while its APIs favor composition over fixed `data` and `actions` objects.

## Composition first

Each block has a root provider and named subcomponents. Put factual values on the subcomponent that displays them, share callbacks and formatters through the root context, and replace interface elements with children.

```tsx
<EventConfirmation onShare={(platform) => share(platform)}>
  <EventConfirmation.Header orderNumber="#EVENT-2048" />
  <EventConfirmation.Details
    eventTitle="MCP Apps Summit"
    eventDate="July 24 · 10 AM"
    eventLocation="Bengaluru"
  />
  <EventConfirmation.Organizer name="mcpcn community">
    <VerifiedBadge />
  </EventConfirmation.Organizer>
  <EventConfirmation.Share />
</EventConfirmation>
```

Omit the root children when you want a polished quick-start composition.

## Registry

The registry includes 21 blocks across payment, events, social, forms, blogging, lists, status, and miscellaneous categories. Every block includes:

- A typed compound API
- Contextual shared behavior
- Arbitrary child composition
- Responsive semantic Tailwind styles
- Quick-start and composed examples

Install a block directly:

```bash
npx shadcn@latest add https://mcpcn.dev/r/event-confirmation.json
```

Browse the block categories in the documentation sidebar or open the full registry at `/r/registry.json`.

## Why copy/paste?

The shadcn model gives your application ownership of the component code. You can adapt layout, behavior, and product language without waiting for a package API to anticipate your use case.
