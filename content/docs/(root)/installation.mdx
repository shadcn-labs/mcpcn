## Prerequisites

mcpcn targets React applications configured for shadcn/ui and Tailwind CSS. Initialize shadcn first if your project does not already have a `components.json` file.

## Install a block

Use the block’s registry URL:

```bash
npx shadcn@latest add https://mcpcn.dev/r/event-confirmation.json
```

The installer copies the block, its category types, and the small compound-context helper into your application.

## Use the default composition

Every root renders a ready-to-use default when no children are supplied:

```tsx
import { EventConfirmation } from "@/components/ui/event-confirmation";

export function Confirmation() {
  return <EventConfirmation onViewTickets={() => openTickets()} />;
}
```

## Compose the slots

Supply named children when your interface needs different content or structure:

```tsx
<EventConfirmation onFollowOrganizer={() => follow()}>
  <EventConfirmation.Header
    heading="You’re on the list"
    orderNumber="#EVENT-2048"
  />
  <EventConfirmation.Details
    eventTitle="MCP Apps Summit"
    ticketCount={1}
    recipientEmail="builder@example.com"
  />
  <EventConfirmation.Organizer name="mcpcn community">
    <VerifiedBadge />
  </EventConfirmation.Organizer>
</EventConfirmation>
```

Subcomponents must remain inside their root. Rendering one outside its provider throws a clear runtime error.

## Local development

To work on this registry itself:

```bash
pnpm install
pnpm typecheck
pnpm registry:build
pnpm dev
```

Run `pnpm build` before deployment to regenerate the registry and verify the documentation site.
